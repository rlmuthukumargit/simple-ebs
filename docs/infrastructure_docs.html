Resource Specification

Project: Simple Terraform Learning | Reference: AWS Beanstalk + CloudWatch Modular Stack

01. Compute Platform (Elastic Beanstalk)

aws_elastic_beanstalk_application
Internal Requirement
Service-level lifecycle management. Specifically set to max_count = 100 to prevent S3 version sprawl.
Technical Why
Provides a unified namespace for multi-environment (Dev/QA) deployment consistency.
aws_elastic_beanstalk_environment
Network Isolation
Requirement: ELBScheme = "internal". This prevents exposure to public subnets and forces all traffic through corporate entry points.
Deployment Strategy
Configured as AllAtOnce for the Dev tier to allow rapid "Un-wedge" recovery during failure-prone CI/CD testing phases.
Health Mechanism
Targets /actuator/health. Required to validate application context readiness (Hibernate/Oracle pool) before ALB traffic registration.
Resource Management
Force-set JAVA_OPTS=-Xmx512m to ensure JVM stability on lower-tier t3a.medium instances.

02. Observability & Telemetry

aws_cloudwatch_log_group
Data Retention
Why created: To bypass Beanstalk's default log destruction. Pre-allocation ensures 30-day persistence for audit compliance.
aws_cloudwatch_log_metric_filter
Pattern Logic
Specifically implements ORA- pattern matching. Required to detect silent database connectivity failures that don't trigger HTTP 5xx codes.
aws_cloudwatch_metric_alarm
Dimension Logic
Target: EnvironmentName.
Critical Requirement: Metrics in CloudWatch are identified by their dimensions. Without the EnvironmentName dimension, the alarm cannot uniquely identify which environment instance is reporting data.
Stability Factor: Using this dimension abstracts the alarm from the underlying EC2 Instance IDs or ASG names, which change during scale-out or deployment events.
Latency SLA
Requirement: 95th Percentile (ApplicationLatencyP95) monitoring. Essential for detecting performance degradation before service-wide timeouts occur.

03. Identity & Permissions

aws_iam_policy_document (SNS)
Security Requirement
Explicitly grants sns:Publish to cloudwatch.amazonaws.com. Without this scoped policy, Alarms fail to reach the email notification layer.
